Identification of the topic


Our idea was to search for information on Palazzo d'Accursio within ArCo and identify any gaps or areas that could be enriched. Before starting the research, we wrote an ASK query to verify the existence of Palazzo d'Accursio in the ArCo database.


PREFIX arco: <https://w3id.org/arco/ontology/arco/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
ASK
WHERE {
?person rdfs:label "Palazzo d'Accursio".
}


Then, we verified whether a picture of Palazzo d'Accursio was present with an OPTIONAL pattern.

PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX arco: <https://w3id.org/arco/ontology/arco/>
PREFIX a-cd: <https://w3id.org/arco/ontology/context-description/>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT *
WHERE {
?cp a arco:ArchitecturalOrLandscapeHeritage ;
rdfs:label ?l .
FILTER(REGEX(?l, "palazzo d'accursio", "i"))
FILTER(REGEX(?l, "bologna", "i"))
OPTIONAL { ?cp foaf:depiction ?depiction }
}


Extraction of IRI and information


After verifying the existence of Palazzo d’Accursio and examining related images, we formulated a query to retrieve its exact IRI.

PREFIX arco: <https://w3id.org/arco/ontology/arco/>
SELECT DISTINCT ?cp
WHERE {
?cp a arco:ArchitecturalOrLandscapeHeritage ;
rdfs:label ?l .
FILTER(REGEX(?l, "Palazzo d'Accursio", "i"))
}
LIMIT 10


This led us to the IRI of Palazzo d’Accursio: https://w3id.org/arco/resource/ArchitecturalOrLandscapeHeritage/0800262039. From there, we began exploring its ontologies and the various pieces of information associated with it.

COMMENTS ON THE 3 DIFFERENT TRIPLES AND BEHAVIOUR OF LLMS


ChatGPT’s triple is the longest, both in terms of number of triples and total character count, because it includes an additional property (l0:name), uses fully qualified URIs (not compacted) and uses language-tagged literals, which add extra characters. The triple proposed by Gemini seems to present some issues like wrong namespace since it uses https://w3id.org/arco/resource/ as a prefix (which is usually for individuals, not ontologies). Another issue could be the Inconsistent use of terms, infact dct:isPartOf is okay but may not align with ARCO's internal ontology (core:isPartOf would be better). Another issue is present in the query proposed by Mistral because it may assume that the resource is a CulturalProperty — but that's unclear if "Agent" actually refers to a physical hall or a person (name "Agent" might be misleading).

Gap 2: decorations and statues


We noticed that the ArCo entry for Palazzo d’Accursio is missing many details about the artistic heritage found in its various rooms. To begin enriching this information, we focused on the presence of frescoes, statues and libraries within the palace. For this task, we employed large language models (LLMs), using all the prompting techniques: zero-shot, few-shot, and chain-of-thought.

COMMENTS ON THE 3 DIFFERENT TRIPLES AND BEHAVIOUR OF LLMS


The first triple, proposed by Chatgpt, states that the building has a distinct physical object (DecorativeApparatus) identified by a URI. It creates a distinct resource for the fresco and linking it via core:hasPart.
The second model, stated by Mistral, uses a blank node ([...]) instead of a URI to represent the decorative apparatus. Conceptually, it is very similar to the first case, but it does not assign a persistent identifier to the fresco: we cannot easily reuse or link this resource elsewhere.
In Gemini’s model there is no longer a separate “fresco” resource. The building is directly linked to the type of decorative apparatus (fresco) via a property (hasInternalDecorativeApparatusOrDecorativeCycle). However, this property is not part of the standard ArCo ontology, suggesting a looser interpretation or hallucination.
